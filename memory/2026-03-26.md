# 2026-03-26: Dark Mode Toggle

## Summary

Added a three-state dark mode toggle (System / Light / Dark) to the home page header. The toggle cycles through system auto-detection, explicit light, and explicit dark modes, persisting the user's choice in `localStorage` under key `nl_theme`.

## Changes

- **style.css**: Added `html[data-theme="dark"]` and `html[data-theme="light"]` CSS variable overrides alongside the existing `@media (prefers-color-scheme: dark)` media query. Added `.theme-toggle` button styles matching the existing header link aesthetic.
- **app.js**: Added internal `Theme` module (getStored, setStored, getEffective, apply, cycle) and `initThemeToggle()` public method. Added sun/moon/monitor SVG icons. Updated card color picker default to use `Theme.getEffective()` instead of raw media query check.
- **index.html**: Added inline FOUC-prevention script in `<head>`, toggle button in `header__right`, and `App.initThemeToggle()` call.
- **newsletter.html, view.html, bookmarks.html**: Added inline FOUC-prevention script in `<head>` only (no toggle button per user request).
- **templates/**: All 6 template files mirrored from root.
- **ARCHITECTURE.md**: Documented Theme Management section.
