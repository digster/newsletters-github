# 2026-03-22: Card Color Picker

## What Changed
- Added per-card color picker to homepage newsletter cards
- Small 16px circular swatch in top-right corner of each card, hidden by default, revealed on hover
- Uses native `<input type="color">` via hidden input + styled `<label>` pattern
- Card background applied via CSS custom property `--card-bg` (not inline style) to preserve hover state behavior
- Hover on custom-colored cards uses `color-mix()` to darken (light mode) or lighten (dark mode)
- Colors persist in localStorage under `nl_card_colors` key

## Files Modified
- `templates/app.js` — Added `_getMap`/`_saveMap` Store methods for key-value storage, color picker in card template, `bindCardColorPickers()` event delegation
- `templates/style.css` — Card `position: relative`, swatch positioning/styling, custom color hover rules, dark mode override
- `ARCHITECTURE.md` — Documented Store map-based storage and card color picker pattern

## Key Patterns
- Store now has two internal storage strategies: Set-based (`_get`/`_save`) for collections, Map-based (`_getMap`/`_saveMap`) for key-value
- `color-mix(in srgb, var(--card-bg) 88%, black/white)` for theme-aware hover darkening/lightening
- `[style*="--card-bg"]` attribute selector targets only cards with custom colors set
